# 智能体指令 (WAT框架与Graphify)

你正运行于 **WAT框架** (Workflows, Agents, Tools) 中。核心原则：AI负责逻辑推理，代码负责确定性执行。

## WAT 架构
1. **Workflows (工作流)**: 位于 `workflows/` 的 Markdown SOP (标准作业程序)。定义目标、输入、工具、输出及边缘情况处理。
2. **Agents (智能体/你)**: 核心决策者。负责阅读工作流，按正确顺序调用工具，优雅处理失败并适时提问。切勿直接执行具体任务（例如：勿直接爬取网页，应阅读相关 SOP 并调用指定的 python 工具）。
3. **Tools (工具)**: 位于 `tools/` 的 Python 脚本，负责实际执行任务。所有凭据和 API 密钥均存放在 `.env` 中。

## 操作指南
1. **复用优先**: 编写新脚本前，先检查 `tools/` 目录寻找可用工具。仅当无适用工具时才创建新脚本。
2. **错误适应与学习**: 遇到报错时，阅读完整堆栈跟踪，修复脚本并重试（若涉及付费API，重试前需先征求我的同意）。将学到的经验（如速率限制、接口特性等）记录更新至工作流中。
3. **维护工作流**: 随着实践不断完善工作流。但除非得到明确指令，否则不得随意新建或覆盖工作流。

## 自我演进循环
每一次失败都是强化系统的机会：识别错误 -> 修复工具 -> 验证修复 -> 更新工作流 -> 继续执行。

## 文件与目录规范
- 最终交付物必须存入云端服务（如 Google Sheets 等），本地文件仅作临时处理。
- `.tmp/`: 临时文件目录（如下载的数据、中间产物），可随时丢弃并重新生成。
- `tools/`: Python 执行脚本。
- `workflows/`: Markdown SOP 指令。
- `.env`: API 密钥（严禁将密钥存入其他任何位置）。
- `credentials.json`, `token.json`: Google OAuth 凭证（已被 gitignore）。

## Graphify 整合 (图谱分析)
在回答任何关于代码库或架构的问题之前，务必先查询 graphify 知识图谱 (使用 `/graphify query` 或查阅 `graphify-out/graph.json`)。在对代码库进行修改后，务必更新图谱。
